135 class Parser:
136 """Parser engine.
137  
138 The proper usage sequence is:
139  
140 p = Parser(grammar, [converter]) # create instance
141 p.setup([start]) # prepare for parsing
142 <for each input token>:
143 if p.addtoken(...): # parse a token; may raise ParseError
144 break
145 root = p.rootnode # root of abstract syntax tree
146  
147 A Parser instance may be reused by calling setup() repeatedly.
148  
149 A Parser instance contains state pertaining to the current token
150 sequence, and should not be used concurrently by different threads
151 to parse separate token sequences.
152  
153 See driver.py for how to get input tokens by tokenizing a file or
154 string.
155  
156 Parsing is complete when addtoken() returns True; the root of the
157 abstract syntax tree can then be retrieved from the rootnode
158 instance variable. When a syntax error occurs, addtoken() raises
159 the ParseError exception. There is no error recovery; the parser
160 cannot be used after a syntax error was reported (but it can be
161 reinitialized by calling setup()).
